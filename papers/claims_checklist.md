# Claims Checklist

Use this file as the paper-facing mirror of `research/contract/05_paper_claims_checklist.md`. Nothing here should get ahead of the evidence.

## Claim Tracking Rules

- No claim without a concrete artifact path in `results/`.
- No proposed-method claim before baseline reproduction.
- Keep observation and interpretation separate.
- If evidence is partial, confidence must stay low and the caveat must say why.

## Paper-Ready Wording

- Supported: "On an adapted AgentPoison full-ReAct comparator under MiniMax, FlowFence-Lite reduces exposed poisoned retrieval and attack manifestation to zero across three repeated runs."
- Supported: "Quarantine is sufficient for the safety effect on this axis; canonical ReAct action writeback improves trajectory hygiene and clean utility stability."
- Supported: "A same-axis safe-view rewrite-only weak defense also blocks exposed poisoned retrieval on this detector-mediated axis, but the selected quarantine-actioncanon variant has stronger clean utility and clearer containment semantics."
- Supported: "An independent static keyword filter also blocks this known-trigger attack on the adapted AgentPoison axis, so FlowFence-Lite should be positioned as structured containment rather than as uniquely necessary for this trigger-string setting."
- Supported, stress-test only: "When the poisoned guidance is paraphrased away from the old static-filter phrases, the non-oracle static keyword filter fails to intervene across three stress-test runs, while FlowFence-Lite still contains exposed poisoned retrieval."
- Supported: "On a fixed 10-question same-axis measured-overhead slice, FlowFence-Lite inspection time is negligible relative to LLM latency, and the quarantine-actioncanon condition did not increase wall-clock time or provider-reported token use versus no defense."
- Supported: "Existing 25-question same-axis traces show moderate overhead proxies for the selected method; this proxy result should be reported separately from the measured 10-question slice."
- Not supported: "FlowFence-Lite reproduces and beats official AgentPoison."
- Not supported: "FlowFence-Lite is broadly better than AgentDojo native defenses."
- Not supported: "FlowFence-Lite outperforms all independent weak defenses on this axis."
- Not supported: "FlowFence-Lite has broad held-out attack or topology generalization evidence."
- Not supported: "FlowFence-Lite significantly improves utility."
- Not supported: "FlowFence-Lite is generally faster or always uses fewer tokens."
